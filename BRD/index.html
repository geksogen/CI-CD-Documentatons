Взаимодействие клиента с системой (юридическое лицо)

На схеме представлены следующие бизнес-процессЫ:

  • Регистрация клиента
graph TD
    Start[Начало] --> CheckEligibility{Проверка
подходящих условий}
    CheckEligibility -- Нет --> NotEligible[Неподходящие
условия]
    CheckEligibility -- Да --> EnterDetails[Ввод
персональных данных]
    EnterDetails --> ValidateDetails{Проверка данных}
    ValidateDetails -- Неверные --> InvalidDetails[Неверные данные]
    ValidateDetails -- Верные --> VerifyEmail[Подтверждение
по Email]
    VerifyEmail -- Ошибка --> ResendEmail[Отправить
повторно]
    VerifyEmail -- Верно --> VerifyPhone[Подтверждение
по Телефону]
    VerifyPhone -- Ошибка --> ResendCode[Отправить
код повторно]
    VerifyPhone -- Верно --> ConfirmDetails[Подтверждение
персональных данных]
    ConfirmDetails --> UploadDocuments[Загрузка
документов]
    UploadDocuments -- Неполные --> IncompleteDocuments[Неполные документы]
    UploadDocuments -- Полные --> Review[Рассмотрение
документов]
    Review -- Отклонено --> Reupload[Переотправка
документов]
    Review -- Одобрено --> CompleteRegistration[Успешная
регистрация]
    IncompleteDocuments --> Reupload
    Reupload --> UploadDocuments
    ResendEmail --> VerifyEmail
    ResendCode --> VerifyPhone
    NotEligible --> End[Завершение]
    CompleteRegistration --> End
